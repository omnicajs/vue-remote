## Scope

На этой странице зафиксированы текущие ограничения поведения `@omnicajs/vue-remote` по сравнению с нативным использованием Vue в одном DOM-контексте.

## Текущие ограничения

1. Вызов разрешенных методов компонентов требует явного `invoke`; более высокоуровневого декларативного API для вызова методов нет.
2. Прямая императивная установка props на смонтированных узлах ограничена и не является универсально безопасной; основной путь это reactive re-render flow.
3. Scoped slot props от host-компонентов не доставляются в remote-слоты.

## Дополнительные ограничения, которые нужно учитывать

1. Refs на native элементы в remote-шаблонах указывают на proxy-узлы remote-дерева, а не на реальные browser DOM-элементы.
2. Remote `nextTick()` дожидается соответствующего host commit и reject-ится, если teardown произошел раньше этого commit.
3. Транспортные гарантии (ordering, retry, backpressure, reconnect semantics) задаются выбранным адаптером/runtime, а не только core API.
4. Ошибки на границе runtime нормализуются, поэтому исходный stack context может сокращаться.
