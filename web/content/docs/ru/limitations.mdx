## Scope

На этой странице зафиксированы текущие ограничения поведения `@omnicajs/vue-remote` по сравнению с нативным использованием Vue в одном DOM-контексте.

## Текущие ограничения

1. Native `v-model` на элементах не поддерживается как first-class поведение на границе runtime.
2. Вызов разрешенных методов компонентов требует явного `invoke`; более высокоуровневого декларативного API для вызова методов нет.
3. Прямая императивная установка props на смонтированных узлах ограничена и не является универсально безопасной; основной путь это reactive re-render flow.
4. `nextTick()` на remote не гарантирует, что host UI уже завершил commit обновлений.
5. Scoped slot props от host-компонентов не доставляются в remote-слоты.

## Дополнительные ограничения, которые нужно учитывать

1. Refs на native элементы в remote-шаблонах указывают на proxy-узлы remote-дерева, а не на реальные browser DOM-элементы.
2. Нет публичного remote-side await примитива, который подтверждает host render commit для конкретного обновления.
3. Транспортные гарантии (ordering, retry, backpressure, reconnect semantics) задаются выбранным адаптером/runtime, а не только core API.
4. Ошибки на границе runtime нормализуются, поэтому исходный stack context может сокращаться.
